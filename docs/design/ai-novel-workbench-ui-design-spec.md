# AI 小说创作工作台 UI 设计规格

版本：v0.1
日期：2026-04-28
用途：作为产品、设计、前端开发、测试验收的统一 UI 依据

## 1. 设计目标

AI 小说创作工作台是一个长时间使用的创作型生产力工具。界面应当安静、稳定、清晰，帮助作者专注写作，同时让复杂信息，例如人物、冲突、伏笔、知识库和质量报告，能够被快速理解和调用。

本 UI 设计规格用于减少产品设计与开发实现之间的差异。开发时应优先遵循本文中的布局、组件、颜色、字体、状态和交互规则。

## 2. 设计原则

### 2.1 专注写作

正文编辑区域必须始终是视觉中心。导航、AI 建议、人物设定、知识库引用等信息都应服务于写作，不应喧宾夺主。

### 2.2 信息密度适中

这是工作台，不是营销网站。界面应支持扫描、比较和重复操作。避免大面积装饰、夸张卡片、巨型标题和过多留白。

### 2.3 明确的上下文

用户在任意页面都应知道：

1. 当前在哪个小说项目中。
2. 当前处于哪个模块。
3. 当前操作对象是什么，例如某章、某个人物或某条伏笔。
4. AI 当前参考了哪些上下文。

### 2.4 可控的 AI

AI 不应表现为不可预测的黑盒。每次生成、改写、评分、总结都要展示输入依据、执行状态和结果去向。

### 2.5 一致性优先

同类操作必须使用同类组件。例如生成类操作统一使用主按钮，编辑类操作统一使用次按钮，危险操作统一使用红色确认弹窗。

## 3. 产品视觉定位

### 3.1 风格关键词

1. 安静
2. 专业
3. 文学感
4. 高信息可读性
5. 编辑器优先
6. 轻量数据可视化

### 3.2 不采用的风格

1. 不采用营销型首页风格。
2. 不采用强烈霓虹科技风。
3. 不采用大面积渐变背景。
4. 不采用复杂玻璃拟态。
5. 不使用 emoji 作为功能图标。
6. 不使用过度圆角的软萌风格。

### 3.3 推荐设计语言

推荐使用“现代编辑工作台”风格：

1. 左侧深浅适中的导航栏。
2. 中间大面积白色或浅色写作区域。
3. 右侧上下文辅助面板。
4. 使用细边框和轻阴影区分层级。
5. 使用少量强调色表示 AI 操作和当前状态。

## 4. 布局系统

### 4.1 桌面端主布局

桌面端采用三栏布局。

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶部栏：项目名 / 当前章节 / 搜索 / 同步状态 / 用户入口        │
├──────────────┬──────────────────────────────┬────────────────┤
│ 左侧导航      │ 主工作区                       │ 右侧上下文面板   │
│ 240px        │ 自适应                         │ 360px          │
│              │                              │                │
└──────────────┴──────────────────────────────┴────────────────┘
```

推荐尺寸：

1. 顶部栏高度：56px。
2. 左侧导航宽度：240px。
3. 右侧上下文面板宽度：360px。
4. 主工作区最小宽度：720px。
5. 页面最小推荐宽度：1280px。
6. 页面最大内容宽度不限制，主工作区随窗口增长。

### 4.2 平板端布局

宽度小于 1024px 时：

1. 左侧导航收起为图标栏，宽度 64px。
2. 右侧上下文面板默认折叠，通过按钮打开为抽屉。
3. 主工作区占据主要空间。

### 4.3 移动端布局

宽度小于 768px 时：

1. 不展示三栏。
2. 顶部展示项目名和当前模块。
3. 底部使用 4 到 5 个主导航入口。
4. AI 上下文面板使用全屏抽屉。
5. 正文编辑器优先占满屏幕。

移动端底部导航建议：

1. 项目
2. 大纲
3. 写作
4. 人物
5. AI

### 4.4 间距系统

使用 4px 基础网格。

```text
space-1 = 4px
space-2 = 8px
space-3 = 12px
space-4 = 16px
space-5 = 20px
space-6 = 24px
space-8 = 32px
space-10 = 40px
space-12 = 48px
```

规则：

1. 组件内部间距优先使用 8px、12px、16px。
2. 模块之间间距优先使用 24px。
3. 页面边距桌面端使用 24px。
4. 移动端页面边距使用 16px。

## 5. 色彩系统

### 5.1 色彩方向

采用低饱和、长时间阅读友好的中性色，配合克制的墨蓝和琥珀色。墨蓝用于 AI 和主操作，琥珀色用于创作灵感、提示和轻量强调。

### 5.2 颜色 Token

```text
color-bg-page        #F6F5F1
color-bg-surface     #FFFFFF
color-bg-subtle      #F0EFEA
color-bg-muted       #E8E5DD

color-text-primary   #23211D
color-text-secondary #5F5A50
color-text-muted     #8A8377
color-text-inverse   #FFFFFF

color-border-light   #E2DED6
color-border-default #D3CDC2
color-border-strong  #B8B0A2

color-primary        #24556F
color-primary-hover  #1D465B
color-primary-active #17384A
color-primary-soft   #E5EEF2

color-accent         #B8822F
color-accent-soft    #F3E8D2

color-success        #2F6B4F
color-warning        #9A6A16
color-error          #B33A32
color-info           #2F5F89

color-ai             #5B4A8B
color-ai-soft        #EAE6F5
```

### 5.3 使用规则

1. 页面背景使用 `color-bg-page`。
2. 主内容面板使用 `color-bg-surface`。
3. 普通边框使用 `color-border-light`。
4. 主按钮使用 `color-primary`。
5. AI 相关标签和状态使用 `color-ai`。
6. 创作提示和灵感类信息使用 `color-accent`。
7. 错误和危险操作使用 `color-error`。
8. 不在业务组件中直接写裸色值，应使用 token。

### 5.4 可访问性要求

1. 正文与背景对比度不低于 4.5:1。
2. 图标按钮必须有 tooltip 和 aria-label。
3. 颜色不能作为唯一信息提示，必须配合文字或图标。

## 6. 字体与排版

### 6.1 推荐字体

中文界面字体：

```text
font-sans:
- Inter
- PingFang SC
- Microsoft YaHei
- system-ui
- sans-serif
```

正文写作字体：

```text
font-writing:
- LXGW WenKai
- Noto Serif SC
- Songti SC
- serif
```

如果第一版不引入额外字体，正文编辑器可使用系统 serif 字体。

### 6.2 字号 Token

```text
text-xs   12px / line-height 16px
text-sm   14px / line-height 20px
text-base 16px / line-height 24px
text-lg   18px / line-height 28px
text-xl   20px / line-height 28px
text-2xl  24px / line-height 32px
text-3xl  32px / line-height 40px
```

### 6.3 使用规则

1. 普通 UI 正文字号使用 14px 或 16px。
2. 编辑器正文字号默认 18px，行高 32px。
3. 页面标题使用 20px 或 24px。
4. 面板标题使用 14px 或 16px，字重 600。
5. 辅助说明使用 12px 或 14px。
6. 不使用小于 12px 的文字。
7. 字距保持默认，不设置负字距。

## 7. 圆角、阴影与边框

### 7.1 圆角

```text
radius-sm = 4px
radius-md = 6px
radius-lg = 8px
radius-xl = 12px
```

使用规则：

1. 普通按钮、输入框、标签使用 6px。
2. 面板和卡片使用 8px。
3. 弹窗使用 12px。
4. 不使用超过 16px 的大圆角。

### 7.2 阴影

```text
shadow-sm = 0 1px 2px rgba(35, 33, 29, 0.06)
shadow-md = 0 8px 24px rgba(35, 33, 29, 0.08)
shadow-lg = 0 20px 48px rgba(35, 33, 29, 0.14)
```

使用规则：

1. 常规工作台布局主要依赖边框，不依赖强阴影。
2. 弹窗、抽屉、悬浮菜单可以使用阴影。
3. 不使用多层彩色阴影。

## 9. 组件规范

### 9.1 按钮

按钮类型：

1. Primary：主要生成、保存、确认。
2. Secondary：编辑、复制、查看。
3. Ghost：工具栏轻量操作。
4. Danger：删除、清空、废弃。
5. AI：生成、总结、评分、改写等 AI 操作。

尺寸：

```text
button-sm height 32px, padding 8px 12px
button-md height 40px, padding 10px 16px
button-lg height 44px, padding 12px 20px
icon-button height 36px, width 36px
```

状态：

1. default
2. hover
3. active
4. focus
5. loading
6. disabled

规则：

1. 每个页面只允许一个最重要的 Primary 操作。
2. AI 操作可使用 AI 色，但不能和保存按钮混淆。
3. loading 状态必须禁用重复提交。

### 9.2 输入框

类型：

1. 单行输入框。
2. 多行文本域。
3. 标签输入。
4. 下拉选择。
5. 搜索框。

规则：

1. 必须有可见 label。
2. 错误信息显示在字段下方。
3. 占位符只用于示例，不替代 label。
4. 长文本字段支持自动增高或固定高度滚动。

### 9.3 面板

面板用于承载功能区域，例如故事圣经区块、人物详情、AI 建议。

结构：

```text
Panel
- Header: 标题 / 简短说明 / 操作
- Body: 内容
- Footer: 可选操作区
```

规则：

1. 不做卡片套卡片。
2. 面板之间使用清晰间距。
3. 面板标题保持短句。

### 9.4 标签

标签用于状态、类型、人物角色、伏笔状态、AI 引用来源。

尺寸：

1. 高度 24px。
2. 左右 padding 8px。
3. 圆角 6px。

常见标签：

```text
主角
反派
盟友
未完成
已完成
已埋设
待回收
AI 生成
知识库引用
需要确认
```

### 9.5 表格

用于人物列表、伏笔列表、知识库条目、质量问题列表。

规则：

1. 表头固定。
2. 行高 44px 或 48px。
3. 支持搜索和筛选。
4. 操作列固定在右侧。
5. 超长文本使用两行截断并提供详情展开。

### 9.6 抽屉

用于移动端和窄屏中的上下文信息。

规则：

1. 右侧抽屉宽度桌面端 420px。
2. 移动端抽屉全屏。
3. 按 Esc 或关闭按钮可关闭。
4. 背景遮罩不应过深。

### 9.7 弹窗

用于确认、批量操作、AI 参数设置。

规则：

1. 宽度 480px、640px、800px 三档。
2. 危险操作必须二次确认。
3. 弹窗中不承载复杂长流程，复杂流程使用独立页面或抽屉。

### 9.8 Toast

用于轻量反馈。

类型：

1. success
2. info
3. warning
4. error

规则：

1. 默认 3 到 5 秒消失。
2. 错误类 toast 保留关闭按钮。
3. 重要错误需要在页面内同步展示，不只依赖 toast。

## 10. 页面 UI 设计

## 10.1 项目列表页

### 页面目标

让用户快速进入已有小说项目，或创建一个新的小说项目。

### 布局

```text
顶部：产品名 / 搜索 / 创建项目
主体：项目列表
侧边：最近打开 / 写作统计
```

### 项目卡片内容

1. 小说名称
2. 题材标签
3. 当前阶段
4. 总字数
5. 最近编辑时间
6. 当前章节
7. 进度条

### 空状态

文案：

```text
还没有小说项目
创建第一个项目，先从一句话创意开始。
```

主操作：创建项目。

## 10.2 项目首页

### 项目首页页面目标

作为项目仪表盘，展示当前创作状态和下一步建议。

### 项目首页布局

```text
顶部：项目名 / 状态 / 当前字数 / 最近保存
第一行：继续写作入口 + 当前章节摘要
第二行：故事进度 / 人物状态 / 伏笔状态 / AI 建议
第三行：最近章节列表 / 最近质量问题
```

### 关键组件

1. 继续写作按钮。
2. 当前章节进度。
3. 伏笔状态统计。
4. AI 编辑提醒。

## 10.3 故事圣经页

### 故事圣经页页面目标

维护小说的顶层设定，作为所有 AI 生成的基础上下文。

### 故事圣经页布局

```text
左：设定目录
中：设定编辑区
右：AI 设定助手
```

### 设定目录

1. 基础信息
2. 主题
3. 世界观
4. 主线冲突
5. 关键规则
6. 时间线
7. 势力

### 交互

1. 每个设定块支持单独保存。
2. 每个设定块支持 AI 扩写、压缩、检查冲突。
3. AI 生成结果默认进入预览，不自动覆盖原文。

## 10.4 人物管理页

### 人物管理页页面目标

创建和维护人物档案，支持作者快速理解人物目标、冲突和成长方向。

### 人物管理页布局

```text
左：人物列表
中：人物详情
右：人物 AI 助手 / 相关关系 / 出场记录
```

### 人物列表项

1. 姓名
2. 角色类型
3. 当前状态
4. 最近出场章节

### 人物详情区

1. 基础资料
2. 目标
3. 恐惧
4. 秘密
5. 欲望
6. 弱点
7. 成长弧线
8. 备注

## 10.5 人物关系页

### 人物关系页页面目标

展示人物之间的关系，支持后续图谱化。

### MVP 设计

第一版优先使用表格和关系详情，不强制做复杂图谱。

### 人物关系页布局

```text
顶部：筛选器 / 新增关系 / AI 发现关系
主体：关系列表
右侧：选中关系详情
```

### 关系详情

1. 人物 A
2. 人物 B
3. 关系类型
4. 关系强度
5. 当前状态
6. 关系变化记录

## 10.6 冲突矩阵页

### 冲突矩阵页页面目标

帮助作者查看故事中的矛盾来源和冲突升级路径。

### 冲突矩阵页布局

```text
顶部：冲突类型筛选 / AI 发现冲突
主体：冲突矩阵表格
右侧：冲突详情
```

### 冲突状态

1. 潜伏
2. 形成
3. 升级
4. 爆发
5. 解决
6. 废弃

### 冲突矩阵页视觉规则

1. 不只用颜色表示状态，必须配文字。
2. 高风险冲突使用 warning 标签。
3. 已解决冲突使用 success 标签。

## 10.7 大纲规划页

### 大纲规划页页面目标

管理全书、分卷和章节大纲。

### 大纲规划页布局

```text
左：卷和章节树
中：当前章节大纲编辑区
右：AI 大纲助手 / 相关设定 / 知识库参考
```

### 章节树

1. 支持折叠分卷。
2. 支持拖拽排序。
3. 支持显示章节状态。
4. 支持快速新增章节。

### 章节大纲表单

字段：

1. 本章目标
2. 出场人物
3. 核心冲突
4. 关键事件
5. 情绪变化
6. 伏笔
7. 结尾钩子

### AI 操作

1. 生成本章大纲。
2. 加强冲突。
3. 增加悬念。
4. 压缩节奏。
5. 生成下一章建议。

## 10.8 章节写作页

### 章节写作页页面目标

这是产品最核心页面。用户在这里基于大纲完成正文写作，并使用 AI 进行生成、续写、改写和质量检查。

### 桌面布局

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶部：项目 / 章节 / 保存状态 / 生成 / 质量检查                │
├──────────────┬──────────────────────────────┬────────────────┤
│ 章节树        │ 正文编辑器                     │ 上下文与 AI     │
│              │                              │                │
└──────────────┴──────────────────────────────┴────────────────┘
```

### 正文编辑器

要求：

1. 默认宽度不超过 760px，保证阅读行长。
2. 居中显示。
3. 字号 18px。
4. 行高 32px。
5. 段落间距 12px。
6. 自动保存状态显示在顶部。
7. 支持字数统计。

### 顶部工具栏

操作：

1. 保存
2. 生成正文
3. 续写
4. 改写选中段落
5. 润色
6. 质量检查
7. 查看版本

### 右侧上下文面板

Tab：

1. 大纲
2. 人物
3. 冲突
4. 伏笔
5. 知识库
6. AI 建议

### AI 生成结果展示

AI 结果不应直接覆盖正文。应先展示在结果面板中，用户可以选择：

1. 插入到光标处。
2. 替换选中内容。
3. 保存为备选版本。
4. 丢弃。


### 伏笔状态

### 表格字段

2. 埋设章节
3. 预计回收章节
4. 当前状态
5. 相关人物
6. 最近更新

## 10.10 知识库页

### 知识库页页面目标

上传参考作品，查看拆分和总结结果，并将写作技巧用于后续创作。

### 知识库页布局

```text
左：知识源列表
中：拆书摘要 / 技巧列表
右：上传状态 / 检索面板
```

### 上传区

状态：

1. 等待上传
2. 上传中
3. 文本提取中
4. 章节拆分中
5. 摘要生成中
6. 入库完成
7. 解析失败

### 版权提示

上传区必须展示简短提示：

```text
请确认你有权上传和私下分析该内容。系统将优先提炼结构、技巧和摘要，不用于复刻原文。
```

## 10.11 质量报告页

### 质量报告页页面目标

展示章节或全书的质量评估结果。

### 质量报告页布局

```text
顶部：选择范围 / 生成报告
主体左：评分概览
主体中：问题列表
主体右：修改建议和快捷操作
```

### 评分维度

1. 情节推进
2. 冲突强度
3. 人物动机
4. 对话自然度
5. 叙事节奏
6. 信息密度
7. 风格一致性
8. 设定一致性
9. 结尾钩子

### 质量报告页视觉规则

1. 评分不要只用雷达图，必须有文字解释。
2. 问题按严重程度排序。
3. 每个问题必须能定位到章节或段落。

## 11. AI 交互设计

### 11.1 AI 操作入口

AI 操作统一使用 `Sparkles` 或 `Wand2` 图标。

按钮文案示例：

1. 生成大纲
2. 生成正文
3. 续写
4. 改写
5. 润色
6. 检查冲突
7. 总结章节
8. 扫描伏笔
9. 生成质量报告

### 11.2 AI 生成状态

状态：

1. 准备上下文
2. 检索知识库
3. 生成中
4. 等待确认
5. 已插入
6. 已丢弃
7. 失败

### 11.3 AI 结果确认

AI 生成结果必须先进入确认区，不自动覆盖用户内容。

确认区操作：

1. 插入
2. 替换
3. 保存为备选
4. 重新生成
5. 丢弃

### 11.4 AI 引用来源

当 AI 使用知识库或项目记忆时，应展示来源，例如：

```text
参考：
- 故事圣经：主线冲突
- 人物：林澈、沈微
- 伏笔：旧铜钥匙
- 知识库：《参考作品 A》的章节结尾技巧摘要
```

## 12. 状态设计

### 12.1 加载状态

1. 页面加载使用骨架屏。
2. AI 生成使用进度步骤。
3. 文件上传使用进度条。
4. 表格加载使用行骨架。

### 12.2 空状态

空状态必须包含：

1. 当前没有什么。
2. 为什么需要它。
3. 下一步操作。

示例：

```text
还没有人物
人物会帮助 AI 保持动机和对话一致。
[创建人物] [AI 生成人物]
```

### 12.3 错误状态

错误状态必须包含：

1. 发生了什么。
2. 是否影响已保存内容。
3. 用户可以做什么。

示例：

```text
AI 生成失败
你的章节内容已保存。可以稍后重试，或减少上下文长度后再次生成。
```

### 12.4 保存状态

章节写作页必须展示保存状态：

1. 未保存
2. 保存中
3. 已保存
4. 保存失败

保存失败时必须有重试入口。

## 13. 响应式设计

### 13.1 Breakpoints

```text
mobile  < 768px
tablet  768px - 1023px
desktop 1024px - 1439px
wide    >= 1440px
```

### 13.2 桌面端

1. 使用三栏布局。
2. 展示完整导航。
3. 右侧上下文面板常驻。

### 13.3 平板端

1. 左侧导航缩为图标。
2. 右侧面板折叠为抽屉。
3. 主内容保持完整。

### 13.4 移动端

1. 使用单栏。
2. 底部导航不超过 5 个入口。
3. 复杂筛选使用底部弹层。
4. 正文编辑器优先显示。

## 14. 可访问性规范

1. 所有交互元素可通过键盘访问。
2. 焦点状态必须可见。
3. 图标按钮必须有 aria-label。
4. 输入框必须有 label。
5. 错误信息必须与字段关联。
6. 普通文本对比度不低于 4.5:1。
7. 大字号文本对比度不低于 3:1。
8. 不禁用浏览器缩放。
9. 动画必须支持 `prefers-reduced-motion`。
10. 表格、抽屉、弹窗必须有合理的焦点管理。

## 15. 动效规范

### 15.1 动效原则

动效只用于解释状态变化，不用于装饰。

### 15.2 时间

```text
micro interaction 150ms
panel open/close  220ms
modal transition  240ms
page transition   180ms
```

### 15.3 缓动

使用 ease-out 或标准 cubic-bezier：

```text
cubic-bezier(0.2, 0, 0, 1)
```

### 15.4 禁止项

1. 不动画 width、height、top、left。
2. 不做循环装饰动画。
3. 不让动画阻塞输入。

## 16. 开发实现约束

### 16.1 Token 优先

开发不得在组件中直接写颜色、字号、圆角和阴影。应使用设计 token。

### 16.2 组件复用

以下组件必须统一封装：

1. Button
2. IconButton
3. TextField
4. TextArea
5. Select
6. Tag
7. Panel
8. Drawer
9. Modal
10. Toast
11. Table
12. EmptyState
13. AiResultPanel
14. ContextPanel
15. ChapterEditor

### 16.3 页面验收

每个页面完成后，需要检查：

1. 是否符合三栏或响应式布局规则。
2. 是否使用设计 token。
3. 是否有 loading、empty、error 状态。
4. AI 结果是否需要用户确认。
5. 是否支持键盘操作。
6. 移动端是否无横向滚动。
7. 文本是否没有溢出。

## 18. UI 开发时间顺序

建议 UI 与功能开发按以下顺序推进。

### UI 阶段 0：设计基础

1. 建立设计 token。
2. 建立基础布局组件。
3. 建立按钮、输入框、标签、面板、弹窗、抽屉、Toast。
4. 建立空状态、错误状态、加载状态组件。

### UI 阶段 1：核心工作台

1. 实现三栏主框架。
2. 实现项目列表页。
3. 实现项目首页。
4. 实现故事圣经页。
5. 实现人物管理页。

### UI 阶段 2：创作主流程

1. 实现大纲规划页。
2. 实现章节写作页。
3. 实现右侧上下文面板。
4. 实现 AI 结果确认面板。
5. 实现保存状态和版本入口。

### UI 阶段 3：复杂记忆管理

1. 实现人物关系页。
2. 实现冲突矩阵页。
3. 实现伏笔管理页。
4. 实现相关筛选和详情抽屉。

### UI 阶段 4：知识库与质量报告

1. 实现知识库上传页。
2. 实现拆书结果页。
3. 实现质量报告页。
4. 实现评分、问题列表和修改建议。

### UI 阶段 5：响应式与打磨

1. 完成平板端折叠布局。
2. 完成移动端单栏布局。
3. 完成键盘可访问性。
4. 完成暗色模式评估，第一版可不实现。
5. 完成视觉验收清单。

## 19. UI 验收清单

开发交付时逐项检查：

1. 是否使用本文定义的颜色 token。
2. 是否使用本文定义的字号、行高和间距。
3. 是否保持工作台风格，而不是营销页风格。
4. 章节写作页是否将正文编辑器作为视觉中心。
5. AI 生成结果是否默认进入确认区。
6. 所有页面是否具备 loading、empty、error 状态。
7. 所有 icon-only 按钮是否有 tooltip 和 aria-label。
8. 移动端是否没有横向滚动。
9. 表格长文本是否不会撑破布局。
10. 弹窗和抽屉是否支持 Esc 关闭。
11. 保存失败、生成失败、上传失败是否有可执行的下一步。
12. 页面是否避免卡片套卡片。
13. 主操作是否明确，且每页不超过一个主要 CTA。
14. 色彩是否保持克制，避免大面积单一蓝紫或强烈渐变。
15. 正文编辑器是否适合长时间阅读和写作。

## 20. 结论

本 UI 设计规格将 AI 小说创作工作台定义为一个安静、专业、长期可用的创作型工作台。后续开发应先实现设计 token 和基础组件，再实现页面。任何页面开发前，都应先对照本文确认布局、组件、状态和交互规则，避免产品设计、UI 设计和前端实现之间出现明显偏差。
